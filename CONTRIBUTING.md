# Contributing to SynQPanel

Thank you for your interest in contributing to SynQPanel!  
Community feedback, ideas, and improvements are welcome and appreciated.

This document outlines how you can participate and what to expect.

---

## Project Scope & Direction

SynQPanel is a GPL-licensed fork focused on:
- Panel-based visualization
- AIDA64 Shared Memory as the sole telemetry source
- Minimal, stable, and maintainable design

Not all feature requests or contributions may align with the project’s goals.
The maintainers reserve the right to accept, modify, or decline contributions to ensure long-term clarity and stability.

---

## Reporting Issues & Feedback

If you encounter a bug or have a suggestion:
- Open an issue on GitHub
- Provide clear steps to reproduce (if applicable)
- Include relevant logs or screenshots when possible

Constructive discussion is welcome.

---

## Submitting Changes (Pull Requests)

If you would like to contribute code:

1. Fork the repository
2. Create a feature or fix branch
3. Keep changes focused and well-documented
4. Submit a pull request with a clear description

Small, targeted changes are preferred over large refactors.

---

## Code Style & Architecture

SynQPanel is built using:
- WPF
- MVVM architecture

Please follow existing patterns and conventions to maintain consistency.

---

This ensures all contributions remain compatible with the project’s license.

---

## Community Conduct

Please keep discussions respectful and constructive.
Disagreements are natural in open source, but personal attacks or hostility are not acceptable.

Thank you for helping improve SynQPanel.
